# Post Studio — Navy & Gold Exact-Fidelity Pass Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Retune the flagship `dark_premium` theme to match the authoritative Navy & Gold spec exactly (Poppins typography, navy radial bg, gold `#C6A274`, gold-rim portrait panels, number-in-pill labels, tooth divider, 3-layer sine wave footer) — and close the P3 final-review Important by resolving Arabic text to the bundled Cairo font.

**Architecture:** This is a *token + theme-gated-builder* pass on the existing P3 theme system. `themes.js` is the single source of truth; only `dark_premium` changes. `render.js` gains theme-flag-gated builders (`label.style==='pill'`, `card.aspect`, `waveFooter.enabled`) so the other three themes are byte-for-byte unaffected, plus a content-driven Arabic→Cairo font resolver that benefits every theme. A new Poppins face is added to the offline `fonts.js` bundle. Export stays client-side foreignObject→canvas with `@font-face` embedded in the export SVG (P2a lesson). No positioning-model rewrite: elements keep fractional positions (P4 owns drag/absolute layout).

**Tech Stack:** Pure-ESM JS modules under `static/post_studio/` (no bundler, no new runtime deps), Python font generator, `node --test` (DOM-free unit), Playwright (`--allow-file-access-from-files`) for render/visual e2e, pytest harness.

## Global Constraints

- **No new runtime dependencies.** Poppins is added as in-repo base64 `@font-face` in `fonts.js` (generated by the existing Python script); no npm/pip additions.
- **render.js stays INLINE STYLES ONLY.** Every visual property serialized into the export is an inline `style` attribute or `setAttribute` on an SVG node. NO `className`/`<style>`/external CSS reaches the `<foreignObject>` export content. (Editor-chrome classNames on never-serialized editor UI are out of scope here.)
- **Custom fonts MUST be embedded as `@font-face` base64 INSIDE the export SVG `<foreignObject>`.** `rasterize.js` already injects `FONT_FACE_CSS`; regenerating `fonts.js` with Poppins keeps that automatic — do NOT touch `rasterize.js`.
- **Only `dark_premium` changes in `themes.js`.** `light_luxury`, `clinical_premium`, `bold_editorial` token objects must be left exactly as-is. New builders are gated on theme flags those three themes do not set.
- **Immutability:** no transform mutates the editable composition; render builds fresh DOM each call.
- **Export canvas square = `[1080, 1080]`** = the spec's reference canvas, so spec px values map 1:1. Portrait `[1080,1350]` / story `[1080,1920]` stay 1080-wide; the wave footer is bottom-anchored so it works at every height.
- **Navy & Gold exact tokens** (verbatim from `docs/superpowers/specs/2026-06-29-template-navy-gold.md`):
  - bg `radial-gradient(circle at 50% 44%, #0C1E3A 0%, #040E20 100%)`
  - gold `#C6A274`; gold-light `#E0C498`; gold-deep `#A08056`; text `#F5F5F0`; muted `#D2D7E1`; navy-panel `#08162C`
  - Poppins: title 78/700 `#F5F5F0`; subtitle 58/700 gold; heading 52; body/label 20/400 `#F5F5F0`; doctor 52/700 gold
  - panel aspect `250 / 320`, radius `14`, border `1px solid rgba(198,162,116,.92)`, no shadow
  - pill height 56, radius `h/2`=28, border `2px solid rgba(198,162,116,.86)`; inner number circle 26px, 2px gold outline; pill text Poppins 20 white, number Poppins 700 19 white
  - divider: two 2px gold lines @86%, `x:18%→tooth-16` and `tooth+16→82%`, tooth icon centered
  - wave footer: 3 layered sines, stroke 2px gold, (amp 14 / freq 0.012 / α38), (amp 18 / freq 0.009 / α30), (amp 11 / freq 0.015 / α22), bottom-anchored
- **Scope deviations (intentional, surfaced):**
  1. **Single-panel pill per block only.** The spec's double-width (516px) pill spanning two panels does not map to our per-block model (each block carries its own label); the flagship before_after (= go.png) needs only single pills. Double-span pill is deferred to P4 deep-customize.
  2. **Arabic→Cairo is content-driven, not `lang`-attribute-driven** (any field can be Arabic in a bilingual designer; detecting Arabic chars per text run is strictly more robust and benefits all four themes). This closes the P3 final-review Important.
  3. **No absolute repositioning.** Elements keep fractional `y`. Matching the spec's exact y-pixels (panels y=360, pills y=708…) is P4's drag-layout concern; fidelity here is tokens + components, judged from the Task 6 screenshots.

```javascript
const px = (n) => `${n}px`;
const setStyle = (el, styles) => { Object.assign(el.style, styles); return el; };

// Arabic-script ranges: any run containing these must render in Cairo (the only
// bundled face with Arabic glyphs) regardless of the theme's Latin display font.
const ARABIC_RE = /[؀-ۿݐ-ݿࢠ-ࣿﭐ-﷿ﹰ-﻿]/;
const FONT_FALLBACK = 'system-ui, "Segoe UI", sans-serif';

function fontStack(font, text) {
  if (text && ARABIC_RE.test(text)) {
    return font ? `"Cairo", "${font}", ${FONT_FALLBACK}` : `"Cairo", ${FONT_FALLBACK}`;
  }
  return font ? `"${font}", ${FONT_FALLBACK}` : 'inherit';
}

function typoStyle(t, text) {
  if (!t) return {};
  return {
    fontFamily: fontStack(t.font, text),
    color: t.color || '#ffffff',
    fontSize: px(t.size || 32),
    fontWeight: String(t.weight || 600),
    letterSpacing: px(t.letterSpacing || 0),
    lineHeight: '1.2',
    margin: '0',
  };
}
```

Add `buildWaveFooter` (place it directly below `buildDivider`). The band is a fixed 140px tall, bottom-anchored; each sine is drawn in the SVG's own `0 0 1080 140` coordinate space:
```javascript
const WAVE_BAND = 140;        // px tall band anchored to the stage bottom
const WAVE_WIDTH = 1080;      // canvas reference width (all sizes are 1080 wide)

function sinePath(baseY, amp, freq, step = 8) {
  let d = `M 0 ${(baseY).toFixed(1)}`;
  for (let x = step; x <= WAVE_WIDTH; x += step) {
    const y = baseY + amp * Math.sin(freq * x);
    d += ` L ${x} ${y.toFixed(1)}`;
  }
  return d;
}

function buildWaveFooter(theme) {
  const wf = theme.waveFooter;
  const svg = document.createElementNS(SVG_NS, 'svg');
  svg.setAttribute('data-ps-wave', '');
  svg.setAttribute('viewBox', `0 0 ${WAVE_WIDTH} ${WAVE_BAND}`);
  svg.setAttribute('preserveAspectRatio', 'none');
  setStyle(svg, {
    position: 'absolute', left: '0', bottom: '0',
    width: '100%', height: px(WAVE_BAND), pointerEvents: 'none',
  });
  // Spec offsets (985/1015/1045 in a 1080 canvas) measured from the bottom,
  // mapped into the local band so the wave is bottom-anchored at any size.
  const baseYs = [45, 75, 105];
  wf.layers.forEach((layer, i) => {
    const path = document.createElementNS(SVG_NS, 'path');
    path.setAttribute('d', sinePath(baseYs[i] ?? 75, layer.amp, layer.freq));
    path.setAttribute('fill', 'none');
    path.setAttribute('stroke', wf.color);
    path.setAttribute('stroke-width', '2');
    path.setAttribute('opacity', String(layer.opacity));
    svg.appendChild(path);
  });
  return svg;
}
```
